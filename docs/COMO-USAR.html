00 — Em uma frase

O essencial

O nosso trabalho mora no GitHub (um "armário online"), organizado em pastas por cliente. Você trabalha em sessões do Claude Code, e o que vale fica salvo nos arquivos — a memória é o Git, não a conversa. Por isso qualquer pessoa continua de onde a outra parou.

02 — Acesso

Como achar os arquivos

Os arquivos não ficam "dentro do chat" — moram no GitHub. Há duas formas:

Para ver / ler

Pelo site do GitHub

Como um Google Drive. Acesse github.com/simpleacc26/simpleacc, clique nas pastas (clientes/ → cliente → estrategia/) e leia. Deixe a branch em main.

Para criar / alterar

Numa sessão do Claude Code

Abra New session no modo Code, selecione Simple Acc → simpleacc → main e diga o cliente na 1ª mensagem. A IA trabalha e abre um PR.

Regra simples: GitHub = ver/ler. Sessão do Claude Code = criar/alterar.

04 — Mãos à obra

Passo a passo para começar

  1. New session (Nova sessão), no modo Code.
  2. No seletor, escolha Simple Acc → simpleacc → main.
  3. Diga o cliente e a tarefa: "Trabalhar em clientes/sense-clinic: criar a landing."
  4. Peça para ler contexto/ e aprendizados.md do cliente antes de criar nada.
  5. Para gerar funil / página / anúncios, use o comando /funil.
  6. Ao terminar, a IA salva e abre um PR → revisar → merge.
  7. Registre o aprendizado em aprendizados.md do cliente.
05 — Comando

O comando /funil

Em qualquer sessão, digite /funil. Ele assume o papel do nosso prompt mestre e conduz: diagnóstico → aprofundamento → resumo → geração (página em HTML puro ou copy de anúncios). Dentro da pasta de um cliente, já usa o contexto dele.

Outros comandos virão (ex.: /roteiro, /copy) conforme cadastrarmos os prompts mestres em prompts/.

07 — Disciplina

Regras de ouro

  • 1. Uma sessão = um cliente/assunto. Nunca misturar dois clientes na mesma conversa.
  • 2. Sempre main no seletor — é a versão oficial.
  • 3. Leia antes de criar: contexto/ e aprendizados.md do cliente.
  • 4. Salve e registre: commit/push + aprendizado no aprendizados.md.
  • 5. Nunca commitar segredos (senhas, tokens). Use .env (ignorado pelo Git).
  • 6. Em dúvida, leia o Manual.
